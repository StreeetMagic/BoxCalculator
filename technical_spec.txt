Техническое задание на веб-приложение «Подбор упаковки»
=====================================================

1. Общие сведения
-----------------
1.1. Назначение: веб-страница, помогающая подобрать подходящую упаковку из заранее заданного каталога по введённым пользователем габаритам и массе изделия.
1.2. Платформа: статическая HTML-страница, работающая в современных браузерах без серверной части.
1.3. Язык интерфейса: русский.

2. Функциональные требования
----------------------------
2.1. Форма ввода должна содержать четыре числовых поля:
    • «Длина, мм» – обязательное поле, принимает положительные целые значения (минимум 1).
    • «Ширина, мм» – обязательное поле, принимает положительные целые значения (минимум 1).
    • «Высота, мм» – обязательное поле, принимает положительные целые значения (минимум 1).
    • «Масса, кг» – обязательное поле, принимает положительные числовые значения (минимум 0.1) с шагом 0.1.
    Поля длины, ширины и высоты используют HTML-атрибут `type="number"` с параметрами `min="1"`, `step="1"` и обязательны (`required`). Поле массы использует `type="number"`, `min="0.1"`, `step="0.1"`, `inputmode="decimal"` и также обязательно.
2.2. Расчёт выполняется автоматически при изменении значений в любом из полей без нажатия дополнительных кнопок и без перезагрузки страницы.
2.3. При расчёте выполняется проверка:
    • введены ли значения в каждое поле;
    • являются ли габариты целыми положительными числами;
    • является ли масса положительным числом.
    В случае некорректного ввода необходимо вывести сообщение «Введите положительные значения. Используйте целые числа для габаритов и число больше нуля для массы.».
2.4. Алгоритм расчёта:
    • К исходной длине и ширине добавляется по 100 мм.
    • К исходной высоте добавляется 225 мм, далее результат округляется вверх до ближайшего шага в 10 мм.
    • Максимально допустимая высота упаковки – 2450 мм. При превышении выводится сообщение об ошибке: «Высота {значение} мм превышает максимум 2450 мм.\nИзмените входные размеры.».
    • Из набора доступных упаковок нужно выбрать варианты, в которые помещаются рассчитанные длина и ширина. Допустима ориентация «как есть» или с перестановкой сторон.
    • Среди подходящих упаковок выбирается упаковка с минимальной площадью основания; при равной площади выбирается упаковка с меньшей суммой длины и ширины.
    • Если подходящая упаковка не найдена, вывести сообщение: «Подходящая упаковка не найдена.\nМинимум по расчёту: {длина} x {ширина} x {высота} мм (Д x Ш x В).».
    • Если упаковка найдена, вывести сообщение: «Разместить на паллете {название} H={высота} мм Масса части {масса} кг», где масса выводится с одним знаком после запятой и использованием запятой в качестве десятичного разделителя.
2.5. Сообщения пользователю выводятся в блок `<pre id="out">`. Рядом с ним располагается кнопка «Скопировать», которая копирует текущий результат в буфер обмена. Кнопка активируется только при наличии валидного результата расчёта.

3. Набор упаковок
-----------------
3.1. Каталог упаковок хранится в массиве объектов `BOXES`, где для каждой упаковки указаны поля `name` (наименование), `L` (длина) и `W` (ширина) в миллиметрах.
3.2. Текущий набор упаковок:
    1) ТДП.004.1.01.022 — 1250 x 850
    2) ТДП.004.1.01.032 — 1800 x 850
    3) ТДП.004.1.01.033 — 1800 x 1225
    4) ТДП.004.1.01.034 — 1800 x 1600
    5) ТДП.004.1.01.042 — 2350 x 850
    6) ТДП.004.1.01.043 — 2350 x 1225
    7) ТДП.004.1.01.044 — 2350 x 1650
    8) ТДП.004.1.01.045 — 2350 x 1950
3.3. Таблица доступных упаковок автоматически строится на основе массива `BOXES`, выводит колонку с наименованием упаковки и показывает значения длины и ширины в отсортированном порядке (большее значение в колонке «Длина, мм»).

4. Пользовательский интерфейс
-----------------------------
4.1. Заголовок страницы – «Подбор упаковки».
4.2. Под формой размещается таблица с перечнем доступных упаковок. Таблица содержит заголовки колонок: «#», «Название», «Длина, мм», «Ширина, мм».
4.3. Таблица заполняется при загрузке страницы, данные формируются скриптом.
4.4. Результирующие сообщения и ошибки отображаются в текстовом блоке `<pre id="out">`, который по умолчанию содержит подсказку «Введите данные для расчёта.».
4.5. Кнопка «Скопировать» располагается рядом с блоком результатов, имеет отключённое состояние при отсутствии готового результата и визуально согласована с остальным интерфейсом.
