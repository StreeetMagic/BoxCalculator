Техническое задание на веб-приложение «Подбор упаковки»
====================================================

1. Общие сведения
-----------------
1.1. Назначение: веб-страница, помогающая подобрать подходящую упаковку из заранее заданного каталога по введённым пользователем параметрам изделия.
1.2. Платформа: статическая HTML-страница, работающая в современных браузерах без серверной части.
1.3. Язык интерфейса: русский.

2. Функциональные требования
----------------------------
2.1. Форма ввода должна содержать четыре числовых поля:
    • «Длина, мм» – обязательное поле, принимает положительные целые значения (минимум 1).
    • «Ширина, мм» – обязательное поле, принимает положительные целые значения (минимум 1).
    • «Высота, мм» – обязательное поле, принимает положительные целые значения (минимум 1).
    • «Масса, кг» – обязательное поле, принимает положительные числовые значения с точностью до сотых (минимум 0,01).
    Поля должны использовать HTML-атрибут `type="number"`, параметр `min="1"` для размеров и `min="0.01"` для массы, а также быть обязательными (`required`).
2.2. Расчёт должен выполняться автоматически при изменении значений в любом из полей без нажатия дополнительных кнопок и без перезагрузки страницы.
2.3. При расчёте выполняется проверка:
    • введены ли значения в каждое поле;
    • являются ли значения длины, ширины и высоты целыми положительными числами;
    • является ли масса положительным числом.
    В случае некорректного ввода размеров необходимо вывести сообщение «Введите положительные целые значения для размеров.».
    В случае некорректного ввода массы необходимо вывести сообщение «Введите положительное значение массы.».
2.4. Алгоритм расчёта:
    • К исходной длине и ширине добавляется по 100 мм.
    • К исходной высоте добавляется 225 мм, далее результат округляется вверх до ближайшего шага в 10 мм.
    • Максимально допустимая высота упаковки – 2450 мм. При превышении выводится сообщение об ошибке: «Высота {значение} мм превышает максимум 2450 мм.\nИзмените входные размеры.».
    • Из набора доступных упаковок нужно выбрать варианты, в которые помещаются рассчитанные длина и ширина. Допустима ориентация «как есть» или с перестановкой сторон.
    • Среди подходящих упаковок выбирается упаковка с минимальной площадью основания; при равной площади выбирается упаковка с меньшей суммой длины и ширины.
    • Если подходящая упаковка не найдена, вывести сообщение: «Подходящая упаковка не найдена.\nМинимум по расчёту: {длина} x {ширина} x {высота} мм (Д x Ш x В).».
    • Если упаковка найдена, вывести сообщение: «Разместить на паллете ТДП.004.1.01.022 H={высота} мм Масса части {масса} кг».
    Значение массы выводится с десятичным разделителем в формате локали `ru-RU`.
2.5. Сообщения пользователю выводятся в заранее определённый блок `<pre id="out">`. Рядом с блоком размещается кнопка «Скопировать», копирующая в буфер обмена всю строку результата при успешном расчёте.

3. Набор упаковок
------------------
3.1. Каталог упаковок хранится в массиве объектов `BOXES`, где для каждой упаковки указаны поля `L` (длина) и `W` (ширина) в миллиметрах.
3.2. Текущий набор упаковок:
    1) 1250 x 850
    2) 1800 x 850
    3) 1800 x 1225
    4) 1800 x 1600
    5) 2350 x 850
    6) 2350 x 1225
    7) 2350 x 1650
    8) 2350 x 1950
3.3. Таблица доступных упаковок должна автоматически строиться на основе массива `BOXES` и показывать значения длины и ширины в отсортированном порядке (большее значение в колонке «Длина, мм»).

4. Пользовательский интерфейс
-----------------------------
4.1. Заголовок страницы – «Подбор упаковки».
4.2. Под формой размещается таблица с перечнем доступных упаковок. Таблица содержит заголовки колонок: «#», «Длина, мм», «Ширина, мм».
4.3. Таблица должна быть заполнена при загрузке страницы, данные должны формироваться скриптом.
4.4. Результирующие сообщения и ошибки отображаются в текстовом блоке `<pre id="out">`, который по умолчанию содержит подсказку «Введите размеры и массу для расчёта.».
